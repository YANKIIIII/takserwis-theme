# Техническая документация: Тема Tak Serwis

## 1. Архитектура и основные принципы
Тема является кастомным решением для WordPress (автосервис), построенным на модульной архитектуре PHP и компонентном подходе к верстке.

*   **Мультиязычность**: Базируется на плагине **Polylang**. Логика шаблонов разделена на `ru` и `pl` через проверки `pll_current_language()`.
*   **Стилизация**: Гибридная система. Основное ядро компилируется из SCSS (`src/scss/`) через **Gulp**. Все компонентные правки, адаптив и экстрагированные из PHP стили находятся в `assets/css/style2.css`.
*   **Сборка**: Gulp используется для компиляции SCSS, Babel JS и генерации WebP. Основной префикс дескрипторов — `callisto`.

## 2. Структура файлов и каталогов
*   `/functions/` — Модульная логика (setup, scripts, styles, post-types, acf).
*   `/assets/` — Скомпилированные ресурсы (CSS, JS, Fonts, Images).
*   `/src/` — Исходники для разработчика (SCSS, JS, RAW images).
*   `/template_parts/` — Переиспользуемые фрагменты (заголовки секций, элементы форм).
*   `front-page.php`, `front-page-new.php`, `service_new.php` — Основные шаблоны лендингов.

## 3. Ядро темы (Logic & Functions)
Критически важные функции сосредоточены в `functions.php` и папке `functions/`:

### 3.1. Кастомные хлебные крошки (`custom_breadcrumbs`)
Сложная функция генерации навигации, которая:
- Учитывает текущий язык (`ru`/`pl`).
- Обрабатывает иерархию страниц и кастомных типов постов (`services`, `deals`).
- Поддерживает кастомные заголовки крошек через ACF поле `breadcrumbs_title`.
- Интегрирована с фильтром Yoast SEO (`fix_care_plan_bc_path`) для коррекции путей услуг.

### 3.2. AJAX-обработка цен (`load_subservices_for_service_callback`)
Динамическая подгрузка прайс-листов для услуг:
- Обрабатывает `POST` запросы с `service_id`.
- Извлекает данные из ACF группы `prices_section` (повторитель `prices`).
- Генерирует HTML-таблицу цен с учетом активного языка.

### 3.3. Автоматизация акций (`check_deal_expiry`)
Функция в `func-post-type.php`, которая при каждой загрузке темы проверяет срок действия постов типа `deals`. Если `expiry_date` меньше текущей даты, пост переводится в статус `draft`.

## 4. Управление контентом (ACF & CPT)
*   **Типы постов**:
    *   `services`: Основные услуги сервиса.
    *   `deals`: Акции и спецпредложения с датой истечения.
*   **ACF Pro**:
    *   **Theme Settings**: Глобальная страница опций для общих настроек темы.
    *   **Flexible Blocks**: Шаблоны страниц строятся на группах полей (Advantages, Team, How we work, Testimonials, Brands).
    *   **Translation**: Тексты кнопок и мелкие строки регистрируются через `pll_register_string` в `func-setup.php`.

## 5. Фронтенд-компоненты
### 5.1. Слайдеры (Swiper.js)
В теме инициализируются три основных типа слайдеров:
- **Swiper1**: Общие контентные слайдеры.
- **Swiper2**: Слайдер команды (адаптивный: 3 колонки -> 1 колонка).
- **Swiper3**: Главный герой-слайдер на главной странице (автоплей 10сек).

### 5.2. Адаптивность и Сетка
- Используется сетка Bootstrap для `row/col` структуры.
- Кастомные брейкпоинты прописаны в `style2.css` (основные: 1200px, 1023px, 767px, 450px).
- Поддержка SVG и автоматическая конвертация в WebP через Gulp.

## 6. Система перевода (L10n)
Для добавления новой строки в интерфейс:
1. Зарегистрируйте её в `functions/func-setup.php` через `pll_register_string()`.
2. Выведите в шаблоне через `<?php echo pll__('Ваша строка', 'context'); ?>`.

## 7. Гайдлайн для разработчика
1.  **CSS**: Категорически запрещено использовать `<style>` в PHP. Все правки — в `style2.css`.
2.  **Шаблоны**: При создании новой секции на главной, добавляйте её параметры в ACF группу `front-page` и создавайте фрагмент в `template_parts/`.
3.  **Изображения**: Используйте WebP для контента. Для иконок — inline SVG или спрайты.
4.  **Безопасность**: Всегда используйте `esc_html()`, `esc_url()` и `wp_kses_post()` при выводе ACF полей.
